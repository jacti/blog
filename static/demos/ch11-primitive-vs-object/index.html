문자열은 진짜 바꿀 수 없을까? SPEC
문자열의 특정 인덱스에 값을 할당해봅니다. 배열과 비교하면 차이가 명확해요.
let str = "hello";
str[0] = "H";          // 수정 시도
console.log(str);         // 바뀌었을까?

let arr = ["h","e","l","l","o"];
arr[0] = "H";             // 수정 시도
console.log(arr.join("")); // 바뀌었을까?
값에 의한 전달 — 원시 값의 독립성 SPEC
원시 값을 복사한 뒤 한쪽만 바꾸면 다른 쪽에 영향이 있을까요?
let a = 10;
let b = a;       // a의 값을 b에 복사
a = 99;           // a만 변경

console.log("a:", a);
console.log("b:", b);
console.log("독립적인가?", a !== b);
참조에 의한 전달 — 객체의 공유 SPEC
객체를 복사하면 어떻게 될까요? 원시 값과 비교해봅시다.
let obj1 = { name: "Kim" };
let obj2 = obj1;          // 참조 복사
obj2.name = "Lee";        // obj2를 통해 수정

console.log("obj1.name:", obj1.name);
console.log("obj2.name:", obj2.name);
console.log("같은 객체?", obj1 === obj2);

3. 실제 구현도 그럴까?

스펙 vs V8 — 메모리 모델 비교 V8
스펙은 "바인딩의 값을 변경하라"고만 말합니다. V8은 값의 종류에 따라 다르게 구현해요.
// 스펙: SetMutableBinding — "Change its bound value to V"
let a = 5;    // 바인딩 a → 값 5
a = 10;        // 바인딩 a → 값 10 (HOW는 정하지 않음)

Environment Record

a → 5

재할당 후

a → 10
스펙은 "어떻게"가 아니라 "무엇이 관찰되어야 하는가"만 정의한다.
// V8 Smi: 작은 정수는 태그드 값으로 슬롯에 직접 저장
// 재할당 = 같은 슬롯 덮어쓰기 (C의 int와 동일)
let a = 5;    // 슬롯[rbp-0x10] = 0x0A (Smi 5)
a = 10;        // 슬롯[rbp-0x10] = 0x14 (Smi 10) ← 같은 주소!

Stack Slot [rbp-0x10]

0x10: 0x0000000A (Smi 5)

재할당 후 (같은 주소)

0x10: 0x00000014 (Smi 10)
Smi는 새 메모리를 할당하지 않는다. 교재의 "새 공간" 모델과 다르다.
// V8 HeapNumber: 큰 숫자/소수점은 힙에 할당
// 재할당 = 새 HeapNumber 생성 + 포인터 변경
let x = 1.5;   // 슬롯 → 힙 0xA000 (HeapNumber 1.5)
x = 2.7;        // 슬롯 → 힙 0xB000 (HeapNumber 2.7) ← 새 주소!

Stack Slot

x → 0xA000
힙 0xA000: 1.5

재할당 후

x → 0xB000
힙 0xB000: 2.7
힙 0xA000: 1.5 (GC 대상)
HeapNumber는 교재의 모델과 일치한다. 새 메모리에 할당하고 주소를 바꾼다.

4. 문자열 인터닝 — 엔진의 비밀 최적화

같은 "hello"인데 내부가 다르다? V8
V8은 문자열 리터럴을 String Table에 등록(인터닝)해서 같은 문자열을 공유해요. 하지만 동적으로 만든 문자열은 인터닝되지 않을 수 있습니다. 아래 코드는 실제 JS를 실행한 결과입니다. 
// 리터럴 vs 동적 생성
const literal = "hello";           // 리터럴 → String Table 등록
const dynamic = "hel" + "lo";      // 동적 연결 → 인터닝 안 됨

console.log(literal === dynamic);  // 값은 같다
// 하지만 내부적으로 같은 메모리를 공유할까?
인터닝 대조표 실험 V8
다양한 방식으로 문자열을 만들어보고, V8에서 --allow-natives-syntax로 측정한 실제 인터닝 결과입니다. 예상해보고 버튼을 눌러 확인하세요.
