      컨텐츠 컴포넌트

      Callout (알림 박스)

      강조하거나 주의를 끌어야 하는 내용을 표시합니다.

      기본 사용법:

      {{< callout type="info" >}}
  여기에 내용을 작성합니다.
{{< /callout >}}

      실제 예시:

      여기에 내용을 작성합니다.

      사용 가능한 타입:

      • info - 정보 (파란색)
      • warning - 경고 (노란색)
      • error - 오류 (빨간색)
      • important - 중요 (보라색)

      고급 옵션:

      {{< callout type="warning" title="주의사항" icon="shield-exclamation" >}}
  커스텀 아이콘과 제목을 사용할 수 있습니다.
{{< /callout >}}

{{< callout emoji="🚀" >}}
  이모지를 아이콘으로 사용할 수도 있습니다.
{{< /callout >}}

      실제 예시:

      커스텀 아이콘과 제목을 사용할 수 있습니다.
      🚀
      이모지를 아이콘으로 사용할 수도 있습니다.

      매개변수:

      • type - 알림 타입 (info, warning, error, important)
      • title - 커스텀 제목
      • icon - 커스텀 아이콘 이름
      • emoji - 이모지 사용
      • collapsed - 기본 접힘 상태 (true/false)

      Cards (카드 레이아웃)

      카드 형태로 콘텐츠를 그리드 레이아웃으로 표시합니다.

      기본 사용법:

      {{< cards >}}
  {{< card link="/docs/page1" title="페이지 1" icon="document-text" >}}
  {{< card link="/docs/page2" title="페이지 2" icon="book-open" >}}
  {{< card link="/docs/page3" title="페이지 3" icon="academic-cap" >}}
{{< /cards >}}

      실제 예시:

      Learning Log
      학습 노트 모음
      Debug Notes
      트러블슈팅 사례
      Playground
      데모 프로젝트

      그리드 컬럼 지정:

      {{< cards cols="2" >}}
  {{< card link="..." title="..." >}}
  {{< card link="..." title="..." >}}
{{< /cards >}}

      카드 옵션:

      {{< card
  link="/docs/guide"
  title="가이드 문서"
  subtitle="자세한 설명"
  icon="book-open"
  tag="NEW"
  tagColor="green"
  image="/images/card-bg.jpg"
  method="resize"
  options="600x q80 webp"
>}}

      카드 매개변수:

      • link - 링크 URL
      • title - 제목
      • subtitle - 부제목
      • icon - 아이콘 이름
      • tag - 태그 텍스트
      • tagColor - 태그 색상 (info, warning, danger, success)
      • image - 이미지 경로
      • imageStyle - 이미지 스타일 (cover, contain 등)
      • method - 이미지 처리 방법 (resize, fit, fill, crop)
      • options - 이미지 옵션 (크기, 품질 등)

      Steps (단계별 가이드)

      순차적인 단계를 표시합니다.

      사용법:

      {{% steps %}}

### 첫 번째 단계

첫 번째 단계에 대한 설명입니다.

### 두 번째 단계

두 번째 단계에 대한 설명입니다.

코드 블록도 포함할 수 있습니다:

\```bash
npm install hextra
\```

### 세 번째 단계

마지막 단계입니다.

{{% /steps %}}

      실제 예시:

      프로젝트 생성

      새로운 Hugo 프로젝트를 생성합니다.

      hugo new site my-site

      테마 설치

      Hextra 테마를 설치합니다.

      로컬 서버 실행

      개발 서버를 시작합니다.

      hugo server -p 1313

      주의사항:

      • {{% steps %}} (퍼센트 기호) 사용 필수
      • 각 단계는 ### (h3) 헤딩으로 구분
      • 마크다운 컨텐츠 포함 가능

      Tabs (탭 인터페이스)

      여러 콘텐츠를 탭으로 전환하며 표시합니다.

      기본 사용법:

      {{< tabs >}}

  {{< tab name="JSON" >}}
  \```json
  {
    "name": "hextra"
  }
  \```
  {{< /tab >}}

  {{< tab name="YAML" >}}
  \```yaml
  name: hextra
  \```
  {{< /tab >}}

  {{< tab name="TOML" >}}
  \```toml
  name = "hextra"
  \```
  {{< /tab >}}

{{< /tabs >}}

      실제 예시:

      {
  "title": "jacti-log",
  "theme": "hextra"
}
      title: jacti-log
theme: hextra
      title = "jacti-log"
theme = "hextra"

      탭 동기화:

      동일한 items를 가진 탭들이 자동으로 동기화됩니다. (hugo.yaml에서 params.page.tabs.sync: true 설정 필요)

      기본 선택 탭:

      {{< tabs defaultIndex="1" >}}
  {{< tab name="JSON" >}}...{{< /tab >}}
  {{< tab name="YAML" >}}...{{< /tab >}}  <!-- YAML이 기본 선택됨 (0부터 시작) -->
  {{< tab name="TOML" >}}...{{< /tab >}}
{{< /tabs >}}

      Details (접기/펼치기)

      접을 수 있는 섹션을 만듭니다.

      기본 사용법:

      {{< details title="더 보기" >}}
  여기에 숨겨진 콘텐츠를 작성합니다.

  - 리스트
  - 코드 블록
  - 다른 마크다운 요소
{{< /details >}}

      실제 예시:

      더 보기

      여기에 숨겨진 콘텐츠를 작성합니다.

      • 리스트 항목 1
      • 리스트 항목 2
      • 코드나 다른 마크다운 요소도 포함 가능

      기본 열림 상태:

      {{< details title="기본 열림" open=true >}}
  기본적으로 펼쳐진 상태로 표시됩니다.
{{< /details >}}

      Filetree (파일 구조)

      파일/폴더 구조를 시각적으로 표시합니다.

      사용법:

      {{< filetree/container >}}
  {{< filetree/folder name="content" >}}
    {{< filetree/folder name="docs" state="open" >}}
      {{< filetree/file name="_index.md" >}}
      {{< filetree/file name="introduction.md" >}}
    {{< /filetree/folder >}}
    {{< filetree/folder name="blog" >}}
      {{< filetree/file name="post-1.md" >}}
    {{< /filetree/folder >}}
  {{< /filetree/folder >}}
  {{< filetree/file name="hugo.yaml" >}}
{{< /filetree/container >}}

      실제 예시:

        • _index.md
        • introduction.md
        • _index.md
        • hextra-components-guide.md
    • hugo.yaml

      • 매개변수:

      • name - 파일/폴더 이름
      • state - 폴더 상태 (open/closed)

      유틸리티 컴포넌트

      Icon (아이콘)

      인라인으로 아이콘을 삽입합니다.

      사용법:

      {{< icon "github" >}} GitHub에서 보기
{{< icon "book-open" >}} 문서 읽기
{{< icon "sparkles" >}} 새로운 기능

      실제 예시:

      GitHub에서 보기 문서 읽기 새로운 기능 3D 오브젝트 코드 작성

      사용 가능한 아이콘:

      • 260+ Heroicons v1 outline 아이콘
      • 전체 목록: Heroicons
      • 주요 아이콘: github, gitlab, codeberg, bitbucket, hugo, jupyter, sparkles, warning, shield-exclamation, book-open, academic-cap, cube, chart-bar, code, document-text, chat 등

      커스텀 아이콘 추가:

      data/icons.yaml에 SVG 경로 추가:

      my-icon: '<svg>...</svg>'

      설정 필요:

      hugo.yaml에 다음 추가:

      enableInlineShortcodes: true

      Badge (배지)

      텍스트 배지를 표시합니다.

      기본 사용법:

      {{< badge content="v1.0" >}}
{{< badge content="NEW" type="success" >}}
{{< badge content="실험적" type="warning" >}}

      실제 예시:

      v1.0
      NEW
      실험적
      중요

      링크가 있는 배지:

      {{< badge link="/docs/intro" content="시작하기" icon="arrow-right" >}}

      매개변수:

      • content - 배지 텍스트
      • type - 색상 타입 (info, warning, error, success)
      • link - 클릭 시 이동할 링크
      • icon - 아이콘 이름

      Include (포함)

      다른 페이지의 콘텐츠를 포함합니다.

      사용법:

      {{< include "path/to/page" >}}

      상대 경로나 절대 경로 모두 사용 가능합니다.

      미디어 컴포넌트

      Asciinema (터미널 녹화)

      터미널 세션 녹화를 재생합니다.

      사용법:

      {{< asciinema file="demo.cast" >}}

      매개변수:

      • file - .cast 파일 경로 (필수)
      • theme - 플레이어 테마 (기본값: “asciinema”, 옵션: “dracula” 등)
      • speed - 재생 속도 배율 (기본값: 1)
      • autoplay - 자동 재생 여부 (기본값: false)
      • loop - 반복 재생 여부 (기본값: false)
      • poster - 재생 전 표시할 프레임 (예: “npt:1:23”)
      • markers - 시간 마커 (예: “1.5:Installation,3.2:Configuration”)

      Jupyter (노트북 임베드)

      Jupyter 노트북을 페이지에 임베드합니다.

      사용법:

      {{% jupyter "notebooks/example.ipynb" %}}
      주피터 임베딩 예시

      What is the Jupyter Notebook?

      The Jupyter Notebook is an interactive computing environment that enables users to author notebook documents that include:

      • Live code
      • Interactive widgets
      • Plots
      • Narrative text
      • Equations
      • Images
      • Video

      These documents provide a complete and self-contained record of a computation that can be converted to various formats and shared with others using email, version control systems (like Git/GitHub) or nbviewer.jupyter.org.

      PDF (PDF 임베드)

      PDF 파일을 페이지에 임베드합니다.

      사용법:

      {{< pdf "https://upload.wikimedia.org/wikipedia/commons/1/13/Example.pdf" >}}

      참고:

      • 로컬 파일: {{< pdf "/files/document.pdf" >}}
      • 원격 URL: {{< pdf "https://example.com/file.pdf" >}}
      • iframe으로 렌더링되며 최소 높이 32rem 적용

      YouTube (유튜브 비디오)

      유튜브 비디오를 임베드합니다 (Hugo 기본 제공).

      사용법:

      {{< youtube "w7Ft2ymGmfc" >}}

      참고:

      • 비디오 ID만 사용 (전체 URL 아님)
      • 예: https://www.youtube.com/watch?v=w7Ft2ymGmfc → "w7Ft2ymGmfc"
      • 추가 옵션: loading="lazy", autoplay="1", class="my-class"
      • 반응형 임베드로 자동 렌더링

      마크다운 기본 기능

      GitHub Alerts (v0.9.0+)

      GitHub 스타일의 알림을 블록쿼트로 작성합니다.

      사용법:

      > [!NOTE]
> 참고 사항입니다.

> [!TIP]
> 유용한 팁입니다.

> [!IMPORTANT]
> 중요한 정보입니다.

> [!WARNING]
> 경고 사항입니다.

> [!CAUTION]
> 주의가 필요합니다.

      실제 예시:

      Note

      참고 사항입니다. 이 문서는 Hextra v0.9.0 이상에서 사용 가능합니다.

      Tip

      유용한 팁입니다. 빠른 참조 테이블을 클릭하면 해당 섹션으로 이동합니다.

      Important

      중요한 정보입니다. 모든 예시는 실제로 렌더링되어 표시됩니다.

      Warning

      경고 사항입니다. Shortcode를 문서화할 때는 이스케이프 처리가 필요합니다.

      Caution

      주의가 필요합니다. 잘못된 문법은 빌드 에러를 발생시킬 수 있습니다.

      Mermaid (다이어그램)

      코드 블록으로 다이어그램을 작성합니다.

      플로우차트:

      ```mermaid
graph TD;
    A[시작] --> B{결정};
    B -->|예| C[실행];
    B -->|아니오| D[종료];
    C --> D;
```

      실제 예시:

        graph TD;
    A[시작] --> B{결정};
    B -->|예| C[실행];
    B -->|아니오| D[종료];
    C --> D;

      Math/LaTeX (수식)

      KaTeX를 사용한 수식 렌더링입니다.

      인라인 수식:

      질량-에너지 등가 공식: $E=mc^2$

      실제 예시 (인라인):

      질량-에너지 등가 공식: E=mc2E=mc^2E=mc2

      피타고라스 정리: a2+b2=c2a^2 + b^2 = c^2a2+b2=c2

      블록 수식:

      $$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$

      실제 예시 (블록):

      ∫−∞∞e−x2dx=π \int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi} ∫−∞∞​e−x2dx=π​f(x)=1σ2πe−(x−μ)22σ2 f(x) = \frac{1}{\sigma\sqrt{2\pi}} e^{-\frac{(x-\mu)^2}{2\sigma^2}} f(x)=σ2π​1​e−2σ2(x−μ)2​

      Shortcode 타입 선택

      • {{< shortcode >}} - HTML 컨텐츠용 (마크다운 처리 안됨)
      • {{% shortcode %}} - 마크다운 컨텐츠용 (마크다운 처리됨)

      예시:

      <!-- HTML만 포함 -->
{{< callout >}}
  <strong>굵은</strong> 텍스트
{{< /callout >}}

<!-- 마크다운 포함 -->
{{% callout %}}
  **굵은** 텍스트
{{% /callout %}}

      실제 예시:

      HTML 모드 (꺾쇠 괄호) - 마크다운이 처리되지 않고 그대로 표시:

      HTML로 굵게 처리하고 마크다운은 그대로 표시됩니다.

      마크다운 모드 (퍼센트 기호) - 마크다운이 렌더링됨:

      HTML로 굵게 처리하고 마크다운도 굵게 렌더링됩니다.

      위 두 예시를 비교하면 **마크다운** 부분의 렌더링 차이를 확인할 수 있습니다.
      현재는 차이가 없습니다.
