# 2차 개발 결과

## 수정 요약
- 홈 랜딩 페이지를 별도 partial(`layouts/partials/home/landing.html`)로 리팩터링하고, `layouts/shortcodes/landing.html`을 통해 `{{< landing >}}` 숏코드로 불러오도록 구조화했습니다.
- 랜딩 배경/커서/파티클 애니메이션이 네비게이션을 침범하지 않도록 `.landing-root`를 네비게이션 높이에 맞춰 고정 배치하고, 뷰포트 전체(헤더 아래)로 확장했습니다.
- 기존 `body` 전역 스타일에 의존하던 CSS를 전부 `.landing-root` 범위로 한정해 다른 페이지 레이아웃이 깨지지 않게 조정했습니다.
- 다크 모드에서 TOC 하단 영역과 사이드 스위처 배경을 투명 처리하고, 사이드바 하단 스위처는 설정(`params.sidebar.display_bottom_switches`)으로만 노출되도록 수정했습니다.
- 메인 네비게이션에는 테마 토글 버튼만 유지하고, `params.editURL`을 비활성화해 상단 “GitHub에서 편집하기” 링크를 제거했습니다.
- 커스텀 아이콘(`layers`, `bug`)으로 인한 빌드 오류를 Hextra에서 제공하는 `cube`, `shield-exclamation`으로 교체했습니다.
- FlexSearch CDN 접근 실패 시 경고만 출력하고 빌드를 계속하도록 `layouts/_partials/scripts/search.html`에 예외 처리를 추가했습니다.

## 테스트
```bash
mkdir -p tmp/hugo_cache
HUGO_CACHEDIR="$(pwd)/tmp/hugo_cache" hugo --gc --minify
```

> 주의: 네트워크 제한 환경에서는 FlexSearch CDN 경고가 남지만 빌드는 성공합니다.
