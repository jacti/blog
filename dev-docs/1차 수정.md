# 1차 수정 내역

## 변경 사항
- `step` 숏코드 누락 오류 해결을 위해 `steps` 컴포넌트 사용 패턴을 최신 Hextra 방식(`{{% steps %}}` + 마크다운 헤딩)으로 일괄 수정했습니다.
- Hextra 테마에 존재하지 않는 `layers`, `bug` 아이콘을 각각 `cube`, `shield-exclamation`으로 교체해 빌드 오류를 제거했습니다.
- `layouts/_partials/scripts/search.html`을 오버라이드하여 FlexSearch 번들을 로컬에서 불러오거나, CDN 접근 실패 시 경고만 출력하고 검색 스크립트를 생략하도록 예외 처리를 추가했습니다.
- `HUGO_CACHEDIR`를 로컬 디렉터리로 지정해 macOS 샌드박스 권한 문제 없이 빌드를 수행할 수 있도록 안내를 포함했습니다.

## 추가 수정 (다크 모드 & 네비게이션 & 홈 레이아웃)
- `assets/css/custom.css`에서 다크 모드 변수를 `html.dark` 컨텍스트로 재정의하고 관련 컴포넌트 클래스 선택자를 보정해 다크 모드 배경/보더 색상이 정상 적용되도록 수정했습니다.
- `layouts/_partials/sidebar.html`을 오버라이드하여 `params.sidebar.display_bottom_switches` 값이 `true`일 때만 언어/테마 스위처를 사이드바 하단에 노출하도록 변경했고, 기본값을 `false`로 두어 플로팅 탭을 제거했습니다.
- `hugo.yaml`의 메인 네비게이션에서 언어 스위처 항목을 테마 토글 메뉴로 교체하고, 사이드바 하단 스위처 비활성화를 위한 설정(`params.sidebar.display_bottom_switches: false`)을 추가했습니다.
- `params.editURL.enable`을 `false`로 설정해 전역 "GitHub에서 편집하기 →" 버튼을 비활성화했습니다.
- 홈 전용 콘텐츠(`content/_index.md`)에 HTML을 직접 삽입해 Hextra 기본 레이아웃을 유지한 채 맞춤 랜딩 디자인을 렌더링하고, 루트의 낡은 `index.html` 파일은 제거했습니다.
- 랜딩 페이지 프론트매터에 `sidebar.exclude: true`, `toc: false`를 추가하고 `layouts/hextra-home.html`을 커스터마이징해 홈 화면에서만 좌우 사이드바를 제거하고 본문을 가운데 정렬했으며, `layouts/_partials/toc.html`의 다크 모드 배경을 투명 처리해 스크롤 버튼 영역이 자연스럽게 blending 되도록 보정했습니다.

## 테스트 코드
아래 명령을 터미널에서 실행해 빌드를 검증하세요. (루트 디렉터리에서 실행)

```bash
mkdir -p tmp/hugo_cache
HUGO_CACHEDIR="$(pwd)/tmp/hugo_cache" hugo --gc --minify
```
