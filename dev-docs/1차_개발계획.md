# 1차 개발 계획

## 현재 파악한 프로젝트 상태
- Hextra 스타터 템플릿 기반으로 구성되어 있으며 커스텀 레이아웃/스타일은 아직 없음.
- `themes/hextra/` 디렉터리가 비어 있어 로컬 빌드 시 테마 자산과 숏코드(`cards`)가 누락된 상태.
- 콘텐츠는 홈/Docs/About 데모 수준이며 실제 섹션은 아직 생성되지 않음.
- 다국어(i18n) 구조는 비어 있고 기본 언어는 한국어로 확장 예정.

## 합의된 정보 구조 및 섹션 목적
- `Learning log`(`/learning-log/`): AI와 함께 정리한 학습 노트를 위키형으로 제공.
- `Debug notes`(`/debug-notes/`): 트러블슈팅 사례 중심의 스토리 아카이브.
- `Playground`(`/playground/`): AI 제작 데모/프로토타입을 카드형으로 열람 후 개별 HTML 페이지로 연결.
- `About`(`/about/`): 자기소개, 경력, 연락처를 통합한 프로필 페이지.
- `Workbench`: `content/workbench/`에 초안과 실험 콘텐츠를 `draft: true`로 유지.

## 레이아웃 및 UI 방향
- `Learning log`, `Debug notes`: Hextra docs 레이아웃(`type: docs`)을 활용해 좌측 사이드바 카테고리 + 우측 TOC를 유지.
- `Playground`: 리스트 페이지는 카드 그리드, 각 데모는 HTML 임베드 또는 별도 레이아웃으로 제공.
- `About`: 히어로 소개 + 경력/스킬 요약 + 연락 섹션 구성.
- 전체 테마 색상은 가독성을 최우선으로 하되 현재 모노톤 기반에서 포인트 컬러를 선택 예정.

## 색상 관련 메모
- 배포본은 흰색/검정 중심으로 보이며 기본 Hextra 팔레트(진한 남청 포인트)가 적용된 것으로 추정.
- 실제 커스텀을 위해선 테마 모듈을 다시 가져와 CSS 변수 기반으로 팔레트 조정 필요.
- 후보 컬러 조합: 슬레이트+틸, 웜그레이+앰버, 인디고+시아 등. 최종 선택은 후속 논의 예정.

## 향후 작업 일정 (우선순위 순)
1. `hugo.yaml`에서 `module.imports`를 활성화하고 `hugo mod tidy`로 테마 자산 동기화.
2. 다국어 설정 추가: `defaultContentLanguage: ko`, `languages` 블록, `i18n/ko.yaml`과 `i18n/en.yaml`에 메뉴/레이블 번역 정의.
3. 콘텐츠 구조 개편: `content/learning-log/`, `content/debug-notes/`, `content/playground/`, `content/workbench/` 생성 및 `_index.md` 템플릿 작성.
4. `Playground` 카드 리스트 레이아웃과 데모 페이지 템플릿 설계 (필요 시 커스텀 파셜 작성).
5. `About` 페이지 개편과 연락 섹션 통합.
6. 선택한 색상 팔레트로 SCSS 커스텀(`assets/css/custom-palette.scss`) 도입 및 라이트/다크 모드 변수 정리.
7. 각 섹션별 샘플 콘텐츠/컴포넌트 구현 후 사용자 피드백 반영.
8. 최종 레이아웃 확정 후 자동 배포 파이프라인에서 `hugo --buildDrafts --buildFuture --minify` 검증 및 배포 절차 정리.

## 참고 사항
- 네트워크 제한으로 실시간 웹 검색은 불가하므로 최신 Hextra 변경 사항은 테마 모듈 소스를 직접 확인하며 반영 예정.
- 빌드 오류(`cards` 숏코드 미존재)는 테마 모듈 복원 후 재확인해야 함.
- 모든 섹션명은 내비게이션에 영어로 노출하고, 설명/콘텐츠는 한국어를 기본으로 작성 후 영어 번역을 추가할 예정.
